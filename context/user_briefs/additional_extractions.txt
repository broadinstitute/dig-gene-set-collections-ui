You are preparing extraction-ready, provenance-rich gene sets for CFDE-REVEAL using the flannick/dig-gene-set-extractors repository.

Overall goals:
- Generate at least one real run per currently supported modality family.
- Generate at least one run for every CFDE DCC that currently has a realistic path to extraction with the existing repo.
- Keep outputs in the shared repo contract:
  - geneset.tsv
  - geneset.meta.json
  - geneset.provenance.json
  - manifest.tsv for grouped outputs
- Always supply a provenance overlay JSON with:
  - source_resource
  - source_dataset_unit
  - landing_page
  - access_route
  - publication_ids if available
  - organism
  - comparison_space_organism if orthologized
  - extractor notes

Global rules:
1. Read the repo docs first and use the documented CLI/workflow for each extractor family.
2. Prefer official public portals, release pages, APIs, or repos over ad hoc mirrors.
3. For each section below, create a folder:
   prototype_inputs/<dataset_slug>/
4. Save:
   - downloaded raw or processed source files
   - a short README.md describing exactly what was downloaded
   - provenance_overlay.json
   - any staged intermediate files needed by the extractor
5. Normalize gene identifiers to what the target extractor expects.
6. Do not invent metadata fields outside the repo contract.
7. After every run, validate the output with the repo validator.
8. Keep a run log:
   prototype_runs/run_log.tsv
   with dataset_slug, extractor, status, output_path, notes.

Priority order:
- First finish the matrix-backed CFDE DCC runs:
  GTEx, MoTrPAC, HuBMAP, SPARC, Bridge2AI, SenNet, 4DN.
- Then finish the modality-only public runs:
  splicing, methylation, CNV, drug response, morphology, calorimetry, ChIP-seq.
- Reuse the same dataset for multiple extractors when appropriate, but emit separate outputs.

   - source_resource: GTEx
   - source_dataset_unit: GTEx open-access bulk tissue expression
   - landing_page: https://www.gtexportal.org/home/datasets
   - access_route: public portal download

Deliverables:
